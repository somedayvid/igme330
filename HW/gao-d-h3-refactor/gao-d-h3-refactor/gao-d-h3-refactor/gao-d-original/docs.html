Process

What do I make?

As with any creative project deciding on a topic is often the most difficult part of the entire project. I didn't want to use any of the provided APIs because they didn't appeal to me, so I scoured through some of the internet services which I utilize and came upon a site that I frequently use for learning Japanese vocabulary called WaniKani. I realized that the website had an API and after looking through the documentation a bit and fidgeting with the documentation a little bit I was able to get a response from the API.

How do I make this?

As I had never used an API to get data before (Save for the GIPHY API assignment ) I had to figure out how to sort through the API data to get the search information that I wanted. The information that the WaniKani API is also already in a JSON form so it was easy enough to read the data, the main problem was searching through it. Since I wasn't too confident on using the xhr and json searching things from the GIPHY assignment with a different API I fell back on something that I was used to doing, which was looping. Though there is a lot of information to loop through on searches (6000 results!) the site still does the process nearly instantly, so maybe a couple thousand isn't actually that intensive for a computer... I also figured that instead of talking to the API each time a search was made, it would be much more efficient to get all the data beforehand, on load, and then just loop and search through the information I already had stored. This didn't impact loading the site either. After that I just had the function create html elements that contained the different information that each entry contains to display the information that is important to each entry.

Endgame

After looking over my website for a bit, I did decide to change up some of my code and substitute a current search feature for a different one. Originally the search was done by filtering by a level from any of the three types on WaniKani. After testing the function a little bit I realized that the function was very limited in its design and the amount of results that came back that it made the site look very sparse in terms of use. I decided to change the specific search by level and definition and made the search more broad so that it is possible to search for the types by either level or defintion so the user can search specifically for a word and also search more broadly within an entire level from the WaniKani database. During this time I was also able to consolidate several seperate functions into one so that I avoid repeating myself. When getting the definition I would loop through and gather and display the same information as for the levels so I was able to use that same function with my new search by level option and also prevent myself from repeating by having the results displayed also go through just one function for both search by methods.

Mistakes

So the data does take a couple seconds to load so pressing the search button immediately will result in nothing being displayed. Though a second press will get the data. Just don't spam it.
